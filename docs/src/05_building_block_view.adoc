[[section-building-block-view]]
== Vista de bloques de construcción

=== Caja blanca: sistema completo

[source]
----
+-------------------------------------------------------------+
|                        AmITraceable                         |
|                                                               |
|  +------------------+        +---------------------------+  |
|  |     webapp       | <----> |          users            |  |
|  |  React + TS SPA  |  HTTPS |  FastAPI (Python)          |  |
|  +------------------+        +---------------------------+  |
|                                     |          |             |
|                              Prometheus     Reddit /         |
|                              (/metrics)     Instagram APIs   |
+-------------------------------------------------------------+
----

Motivación::
Separación clásica frontend/backend. El backend es agnóstico de
plataforma de despliegue (se ejecuta igual en local, en Docker o en el
runner de CI); el frontend es una SPA estática servida por Nginx en
producción.

Bloques contenidos::
[cols="1,2", options="header"]
|===
| Bloque | Responsabilidad

| `webapp`
| Interfaz de usuario: pantalla de consentimiento/selección de
  plataforma y dashboard del informe de exposición.

| `users`
| Lógica de dominio completa: OAuth por plataforma, extracción de datos,
  fingerprinting, inferencia de atributos, scoring, generación del
  informe.

| Prometheus / Grafana
| Observabilidad: métricas HTTP del backend y dashboards.
|===

Interfaces importantes::
* `webapp` -> `users`: API REST/JSON documentada automáticamente por
  FastAPI en `/docs` (OpenAPI).
* `users` -> Reddit/Instagram: APIs OAuth 2.0 oficiales de cada
  plataforma (ver <<section-context-and-scope>>).

==== `auth/{platform}_oauth.py`

Propósito/Responsabilidad::
Flujo OAuth 2.0 completo de una plataforma: redirección a autorización,
canje de `code` por `access_token` (con paso intermedio de canje a token
de larga duración en el caso de Instagram), almacenamiento en cookie de
sesión firmada, y endpoints de `logout`/`status`.

Interfaces::
`GET /auth/{platform}/login`, `GET /auth/{platform}/callback`,
`POST /auth/{platform}/logout`, `GET /auth/{platform}/status`.

Características de calidad::
Sin persistencia; validación de `state` contra CSRF; claves de sesión
con prefijo de plataforma (`reddit_oauth_state`, `instagram_oauth_state`)
para que ambas puedan coexistir sin colisión.

Ubicación::
`users/app/auth/reddit_oauth.py`, `users/app/auth/instagram_oauth.py`.

==== `{platform}_client.py`

Propósito/Responsabilidad::
Extraer los datos públicos de la cuenta autenticada y normalizarlos al
modelo genérico `SocialProfile`/`SocialPost`. Aplica el principio de
minimización de datos: solo se piden los campos necesarios para el
análisis.

Interfaces::
Clase `RedditClient`/`InstagramClient` con método asíncrono
`fetch_profile() -> SocialProfile`.

Ubicación::
`users/app/reddit_client.py`, `users/app/instagram_client.py`.

==== `analysis_router.py`

Propósito/Responsabilidad::
Orquesta el pipeline común (fingerprint -> inferencia -> scoring ->
informe) para cualquier plataforma registrada. Resuelve el cliente
correspondiente a partir de la sesión mediante un diccionario de
factories (`_PLATFORM_CLIENT_FACTORIES`), sin lógica condicional
especial por plataforma en el cuerpo del endpoint. Expone tres variantes
del mismo análisis:

* `POST /api/analyze/{platform}` -- variante síncrona clásica, responde
  con el `ExposureReport` completo al terminar.
* `GET /api/analyze/{platform}/stream` -- misma lógica vía Server-Sent
  Events, emitiendo hitos de progreso reales ("Leyendo publicaciones...",
  "Analizando fotos...", con contadores) a medida que el pipeline los
  completa, para que el frontend pueda mostrar una pantalla de progreso
  en vivo en vez de una espera opaca.
* `POST /api/analyze/ai-summary` -- endpoint aislado del pipeline
  principal: recibe un `ExposureReport` ya generado y lo reenvía a
  `ai_analysis.py`.

Interfaces::
Ver arriba. Todos documentados con sus posibles códigos de respuesta en
`/docs` (OpenAPI).

Ubicación::
`users/app/analysis_router.py`.

==== `nlp/fingerprint.py`

Propósito/Responsabilidad::
Calcula longitud media de frase, riqueza de vocabulario (type-token
ratio), tasa de uso de emojis, distribución horaria de actividad,
comunidades/etiquetas más frecuentes (`top_groups`), keywords (TF-IDF) e
idioma detectado.

Ubicación::
`users/app/nlp/fingerprint.py`.

==== `nlp/attribute_inference.py`

Propósito/Responsabilidad::
Heurísticas explicables (listas curadas + regex) para inferir
ubicación, ocupación y rutina, cada una con nivel de confianza y
evidencia (permalinks).

Ubicación::
`users/app/nlp/attribute_inference.py`.

==== `nlp/demographic_extraction.py`

Propósito/Responsabilidad::
Detecta declaraciones EXPLÍCITAS que el propio usuario hace sobre sí
mismo en su texto ("tengo 24 años", "vivo en León", "estudio Medicina"),
mediante regex, distinto de `attribute_inference.py` (que infiere de
forma indirecta a partir de en qué comunidades participa). Alimenta el
estimador de k-anonimato.

Ubicación::
`users/app/nlp/demographic_extraction.py`.

==== `scoring/k_anonymity.py`

Propósito/Responsabilidad::
Estima, para cada declaración detectada por `demographic_extraction.py`,
cuánta población en España comparte esa combinación de características
(ver <<section-concepts>>, "Estimación de k-anonimato sin base de datos
sintética"). Devuelve una cascada de pasos (`PopulationNarrowingStep`)
con la población restante, nivel de riesgo y fuente (texto/imagen) de
cada uno.

Ubicación::
`users/app/scoring/k_anonymity.py`, tablas en `users/app/data/ine_reference.py`.

==== `vision/geolocation.py` (opcional)

Propósito/Responsabilidad::
Estima la provincia más probable de una foto de Instagram por similitud
visual (DINOv2 + FAISS) contra un índice de imágenes georreferenciadas
de España (ver <<section-concepts>>). Módulo *best-effort*: se degrada a
"sin resultado" si el índice no está construido o faltan sus
dependencias opcionales.

Interfaces::
`estimate_locations_for_posts(posts, progress_callback=None)`, llamado
desde `report/generator.py` solo cuando `platform == "instagram"`.

Ubicación::
`users/app/vision/geolocation.py`, scripts de preparación del índice en
`users/scripts/`.

==== `ai_analysis.py` (opcional)

Propósito/Responsabilidad::
Envía un `ExposureReport` ya generado a Mistral AI y devuelve
conclusiones priorizadas en lenguaje natural. Desacoplado del pipeline
principal: se invoca solo bajo demanda explícita desde
`POST /api/analyze/ai-summary`.

Ubicación::
`users/app/ai_analysis.py`.

==== `scoring/privacy_score.py`

Propósito/Responsabilidad::
Motor de scoring ponderado (0-100): riesgo de geolocalización, datos
inferibles, facilidad de deanonimización. El componente de
"consistencia de identidad entre plataformas" está a 0, documentado
explícitamente como fuera de alcance (no hay correlación entre
plataformas en esta versión).

Ubicación::
`users/app/scoring/privacy_score.py`.

==== `report/generator.py`

Propósito/Responsabilidad::
Ensambla el `ExposureReport` final: fingerprint, atributos inferidos,
score, estrechamiento de población (k-anonimato), puntos de ubicación
por imagen (si aplica), y recomendaciones generadas a partir de los
umbrales del score.

Ubicación::
`users/app/report/generator.py`.

==== `Landing.tsx`

Propósito/Responsabilidad::
Pantalla de consentimiento explícito y selector de plataforma en forma
de mazo de cartas deslizable (Reddit, Instagram, X como "Coming Soon"),
con navegación por arrastre (Pointer Events), flechas y puntos
indicadores.

Ubicación::
`webapp/src/pages/Landing.tsx`.

==== `Dashboard.tsx`

Propósito/Responsabilidad::
Dispara el análisis (vía streaming SSE, con pantalla de progreso en
vivo) y renderiza el informe completo: score global con desglose, tabla
de estrechamiento de población, mapa de ubicaciones estimadas por foto,
atributos inferidos con evidencia, gráfico de actividad horaria, perfil
de escritura, recomendaciones, análisis con IA bajo demanda, y botón de
descarga del informe en JSON. Determina la plataforma activa por query
param (`?platform=reddit|instagram`).

Ubicación::
`webapp/src/pages/Dashboard.tsx`.

==== `PopulationNarrowingTable.tsx`

Propósito/Responsabilidad::
Muestra la cascada de estrechamiento de población (k-anonimato):
información detectada, población restante estimada, nivel de riesgo, y
si la fuente fue texto autodeclarado o una estimación por imagen (con
aviso de menor fiabilidad en ese segundo caso).

Ubicación::
`webapp/src/components/PopulationNarrowingTable.tsx`.

==== `LocationMap.tsx`

Propósito/Responsabilidad::
Mapa (Leaflet + OpenStreetMap, sin API key) con un punto por foto cuya
ubicación se ha podido estimar, con tamaño/color según el nivel de
confianza y tooltip con provincia, confianza y coordenadas exactas al
pasar el ratón.

Ubicación::
`webapp/src/components/LocationMap.tsx`.

==== `AiSummaryCard.tsx`

Propósito/Responsabilidad::
Botón bajo demanda para enviar el informe ya generado a
`POST /api/analyze/ai-summary` y mostrar las conclusiones. Distingue
explícitamente el caso "no disponible" (503: sin API key o cuota
agotada) de un error genérico (con opción de reintentar).

Ubicación::
`webapp/src/components/AiSummaryCard.tsx`.
