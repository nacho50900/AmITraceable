[[section-runtime-view]]
== Vista de tiempo de ejecución

=== Escenario 1: Autenticación y consentimiento (Reddit)

Flujo OAuth 2.0 completo desde que el usuario elige "Conectar con
Reddit" en el mazo de cartas hasta que llega autenticado al dashboard.

[plantuml, "Secuencia OAuth Reddit", png]
----
actor Usuario
participant "webapp\n(Landing)" as FE
participant "users\n(reddit_oauth.py)" as BE
participant "Reddit\n(authorize + token)" as Reddit

Usuario -> FE: Selecciona carta "Reddit" y pulsa "Conectar"
FE -> BE: GET /auth/reddit/login
BE -> BE: genera state (CSRF), lo guarda en sesión
BE --> Usuario: 307 redirect a Reddit (authorize)
Usuario -> Reddit: Pantalla de consentimiento
Reddit --> Usuario: aprueba
Reddit -> BE: GET /auth/reddit/callback?code=...&state=...
BE -> BE: valida state
BE -> Reddit: POST /api/v1/access_token (code -> access_token)
Reddit --> BE: access_token
BE -> BE: guarda access_token en cookie de sesión firmada
BE --> Usuario: 307 redirect a /dashboard?platform=reddit
Usuario -> FE: GET /dashboard?platform=reddit
----

=== Escenario 2: Autenticación y consentimiento (Instagram)

Mismo patrón que Reddit, con un paso adicional: Instagram exige canjear
el token corto por uno de larga duración (60 días).

[plantuml, "Secuencia OAuth Instagram", png]
----
actor Usuario
participant "webapp\n(Landing)" as FE
participant "users\n(instagram_oauth.py)" as BE
participant "Instagram\n(authorize + token)" as IG

Usuario -> FE: Selecciona carta "Instagram" y pulsa "Conectar"
FE -> BE: GET /auth/instagram/login
BE --> Usuario: 307 redirect a Instagram (authorize)
Usuario -> IG: Pantalla de consentimiento (cuenta Business/Creator)
IG --> Usuario: aprueba
IG -> BE: GET /auth/instagram/callback?code=...&state=...
BE -> IG: POST /oauth/access_token (code -> token corto)
IG --> BE: access_token corto + user_id
BE -> IG: GET /access_token?grant_type=ig_exchange_token
IG --> BE: access_token largo (60 días)
BE -> BE: guarda access_token + user_id en cookie de sesión
BE --> Usuario: 307 redirect a /dashboard?platform=instagram
----

=== Escenario 3: Pipeline de análisis (con progreso en vivo)

Ocurre por completo en memoria durante una única conexión; no se escribe
nada a disco en ningún punto. El frontend usa la variante de streaming
(`GET /api/analyze/{platform}/stream`, Server-Sent Events) para mostrar
una pantalla de progreso real en vez de una espera opaca.

[plantuml, "Secuencia de análisis con progreso", png]
----
participant "webapp\n(Dashboard)" as FE
participant "analysis_router" as Router
participant "{platform}_client" as Client
participant "nlp.fingerprint" as FP
participant "nlp.attribute_inference" as Attr
participant "scoring.privacy_score" as Score
participant "report.generator" as Report
participant "nlp.demographic_extraction\n+ scoring.k_anonymity" as KAnon
participant "vision.geolocation\n(solo Instagram, opcional)" as Geo

FE -> Router: GET /api/analyze/{platform}/stream
Router -> Router: busca factory de la plataforma en el diccionario
Router -> Client: construye cliente con el token de sesión
Client -> Client: llama a la API externa (perfil + posts/media)
Client --> Router: SSE: "Leyendo publicaciones..." (progreso)
Client --> Router: SocialProfile (posts normalizados)
Router -> FP: build_fingerprint(posts)
FP --> Router: WritingFingerprint
Router -> Attr: infer_attributes(posts)
Attr --> Router: [InferredAttribute]
Router -> Score: compute_score(posts, fingerprint, attrs)
Score --> Router: PrivacyScore
Router -> Report: generate_report(platform, ..., progress_callback)
Report -> KAnon: extract_demographics(posts) + estimate_population_narrowing(...)
KAnon --> Report: [PopulationEstimate]
alt platform == "instagram"
  Report -> Geo: estimate_locations_for_posts(posts, progress_callback)
  Geo --> Report: SSE: "Analizando fotos..." (progreso, N/M)
  Geo --> Report: [ImageLocationPoint]
end
Report --> Router: ExposureReport
Router --> FE: SSE final: {"done": true, "report": {...}}
----

=== Escenario 4: Análisis del informe con IA (opcional, bajo demanda)

Independiente del escenario anterior: el usuario ya tiene el informe en
pantalla y pulsa explícitamente "Analizar con IA".

[plantuml, "Secuencia de análisis con IA", png]
----
actor Usuario
participant "webapp\n(AiSummaryCard)" as FE
participant "analysis_router\n(ai_summary)" as Router
participant "ai_analysis.py" as AI
participant "Mistral AI API" as Mistral

Usuario -> FE: Pulsa "Analizar con IA"
FE -> Router: POST /api/analyze/ai-summary (ExposureReport ya generado)
Router -> AI: analyze_report_with_ai(report)
AI -> AI: ¿MISTRAL_API_KEY configurada?
alt sin API key
  AI --> Router: AiAnalysisUnavailable
  Router --> FE: 503 (no disponible)
else con API key
  AI -> Mistral: POST /v1/chat/completions (informe en JSON + system prompt)
  alt cuota agotada (429)
    Mistral --> AI: 429
    AI --> Router: AiAnalysisUnavailable (sin reintentar)
    Router --> FE: 503 (no disponible ahora mismo)
  else éxito
    Mistral --> AI: conclusiones en lenguaje natural
    AI --> Router: [conclusiones]
    Router --> FE: 200 {"conclusions": [...]}
  end
end
----

=== Escenario 5: Integración continua (push a `main`)

[plantuml, "Secuencia de CI", png]
----
actor Dev
participant "GitHub" as GH
participant "GitHub Actions\n(build.yml)" as CI
participant "SonarCloud" as Sonar

Dev -> GH: git push (main)
GH -> CI: dispara workflow build.yml
CI -> CI: cache npm (webapp) / pip (users), instala si falta
CI -> CI: npm run test:coverage (webapp) -> lcov.info
CI -> CI: pytest --cov=app --cov-report=xml (users) -> coverage.xml
CI -> Sonar: sonar-scanner (fuentes + coberturas)
Sonar --> CI: resultado del Quality Gate
CI --> Dev: estado del workflow (verde/rojo) en GitHub
----
